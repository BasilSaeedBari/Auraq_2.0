# Auraq 2.0 — Topical Past Paper Compiler

A complete ground-up rewrite of the Auraq past-paper compiler. Auraq 2.0 compiles past exam papers into topical booklets (QP, MS, and Merged) using a **registry-first design**, **vector-preserving PDF extraction**, **highly parallelized downloads**, and **one-shot batch AI classification**.

---

## Key Features & Upgrades in v2.0

*   **Registry-First Architecture:** Each PDF is parsed once to generate a cached `_registry.json` next to it on disk. Sub-part labels like `(a)`, `(b)`, `(i)`, `(ii)` and their spatial boundaries are logged.
*   **Zero Answer-Space Extraction:** The parser detects where question text actually ends (`text_end_y`). Booklets are generated by clipping precisely below the text, leaving out blank answer sheets and write-in areas.
*   **Batch AI Classification:** Instead of making one slow API request per question (causing 12-hour compilations), Auraq 2.0 packs all questions from a paper into a single batch JSON request to Groq. One call per paper.
*   **True Parallel Multiprocessing:** Registry building is CPU-bound. v2.0 leverages python's `ProcessPoolExecutor` with a Windows-safe `spawn` context to parse multiple papers concurrently.
*   **Clean Structured Filesystem:** No more flat temp folders. Files are cached logically:
    `downloads/{subject}/{year}/{session}/Paper_{number}/{QP,MS}/{filename}.pdf`
*   **PapaCambridge Primary Source:** Adds PapaCambridge as the primary high-speed source with fallbacks to BestExamHelp and Dynamic Papers.
*   **Pure PDF Outputs:** Preserves vector math, diagrams, and formatting via PyMuPDF clipping. No lossy DOCX conversions.

## Installation

1. Make sure you have python 3.9+ installed.
2. Clone or open the project folder.
3. Install the package in editable mode:
   ```powershell
   cd Auraq_2.0
   pip install -e .
   ```

---

## Setup API Keys

To use Groq batch classification, set your API key.
1. Copy `.env.example` to `.env` in the root of the project:
   ```powershell
   copy .env.example .env
   ```
2. Open `.env` and fill in your Groq API Key:
   ```env
   GROQ_API_KEY=gsk_your_key_here
   ```
   *Alternatively, you can set the key inside the GUI Preferences window.*

## How to Run

### Graphical User Interface (GUI)
Simply run the package without arguments:
```powershell
py -m auraq2.main
```

### Command Line Interface (CLI)
You can orchestrate the compiler via command line.
```powershell
py -m auraq2.main --curriculum "Cambridge A-Levels" --subject 9709 --paper 1 --variants 1 2 --series s w --start 2023 --end 2025 --topical
```

#### CLI Options:
*   `--curriculum, -c` : Curriculum name (e.g. `"Cambridge A-Levels"`)
*   `--subject, -s` : Syllabus code (e.g. `9709`)
*   `--paper, -p` : Paper code (e.g. `1`, `3`)
*   `--variants, -V` : Space-separated list of variant numbers (e.g. `1 2`)
*   `--series` : Series short-codes: `s` (May/June), `w` (Oct/Nov), `m` (Feb/March), `j` (January)
*   `--start`, `--end` : Compile range years (e.g. `--start 2022 --end 2025`)
*   `--topical` : Enable generating topical booklet folders.
*   `--ai-mode` : `hybrid` (recommended), `batch` (Groq only), or `heuristics` (keywords only)
*   `--verbose, -v` : Show debug logs.

## Configuration Settings
On the first launch, a configuration file is created in your system AppData:
`%APPDATA%/auraq2/config.ini`

You can customize:
*   `sources_order` : Priority order for download servers (defaults to `papacambridge,bestexamhelp,dynamicpapers`).
*   `download_directory` : Root folder where past paper PDFs and registries will be stored.
*   `qp_top_margin`, `qp_bottom_margin` : Header/footer clipping offsets (in points) to filter noise.
*   `batch_confidence_threshold` : Min confidence level below which heuristic keywords override AI responses.
